# 구현 계획 - 비즈테크 클래식 ERP 시스템 재구현 (사진 기반)

제공된 스크린샷의 레이아웃과 감성을 완벽하게 재현하기 위해 처음부터 다시 구축합니다.

## 1. 디자인 가이드라인 (사진 기반)
- **컬러 스키마**: 클래식 ERP 고유의 베이지색 배경(`#f5f2e8`), 연한 회색 툴바, 푸른색 계열의 포인트 컬러 사용.
- **레이아웃 구조 (MDI 스타일)**:
    - 상단: 시스템 정보 및 로그인 정보 (`system-bar`)
    - 상단 하단: 주 메뉴바 (파일, 매입/매출관리 등 `menu-bar`)
    - 중앙: 각 기능을 담당하는 독립된 윈도우 창들이 로드되는 영역 (`workspace`)
    - 윈도우 내부 구도: **[툴바 버튼군] -> [상세 입력 폼] -> [데이터 그리드 표]** 순서로 배치.

## 2. 주요 모듈 및 기능
- **메인 쉘**: 메뉴 클릭 시 해당 기능의 윈도우를 중앙 영역에 표시.
- **매출 등록 (Sales)**: 거래처 선택, 일자, 결제 정보 입력 및 하단 제품별 수량/단가 입력 시 자동 합계 계산.
- **거래처 등록 (Customer)**: 사업자 정보, 담당자 연락처, 주소 등 상세 정보 입력 및 하단 목록 조회.
- **제품 등록 (Product)**: 매입/매출 단가, 5단계 단가(A~E), 분류, 이미지 관리 기능.

## 3. 기술 스택
- **HTML5/CSS3**: 바닐라 CSS로 클래식한 컴포넌트 디자인 (Flexbox 및 Grid 활용).
- **JavaScript**: 창 전환 로직, 실시간 금액 계산, 간단한 데이터 관리.
- **Icons**: Lucide Icons (스크린샷의 아이콘 느낌과 유사한 현대적 아이콘).

## 4. 작업 단계
1. **기반 구축**: `index.html`과 `style.css`에서 전체 레이아웃과 공통 컴포넌트(버튼, 인풋, 표) 디자인.
2. **거래처 등록 모듈**: 사진 `KakaoTalk_20260204_114912544.jpg`의 구도를 100% 재현.
3. **매출 등록 모듈**: 사진 `화면 캡처 2026-02-04 113743.png`의 구도를 재현하고 자동 계산 로직 적용.
4. **제품 등록 모듈**: 사진 `화면 캡처 2026-02-04 113903.png`를 바탕으로 상세 단가 관리 화면 구현.
5. **통합 및 연동**: 메뉴 클릭 시 창 열기/닫기 기능 및 전체 UI 폴리싱.
